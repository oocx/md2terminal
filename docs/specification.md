# Specification

## Purpose

md2terminal is a terminal-based Markdown viewer. It is primarily used to view reports generated by [tfplan2md](https://github.com/oocx/tfplan2md).

## Usage

```
md2terminal <file.md>        # view a file
md2terminal -                # read from stdin
```

## Functional Requirements

### Markdown Rendering
- **FR-01** Render headings (H1–H6) with decorative rule lines for H1/H2.
- **FR-02** Render paragraphs with word-wrap to fit the terminal width.
- **FR-03** Render fenced code blocks with syntax highlighting and a labeled border.
- **FR-04** Render inline code, bold, italic, strikethrough, and links with distinct styling.
- **FR-05** Render blockquotes with a colored left-border gutter line.
- **FR-06** Render unordered and ordered lists.
- **FR-07** Render horizontal rules.
- **FR-08** Render `<details>`/`<summary>` HTML blocks as collapsible sections.
- **FR-09** Render `<pre><code>` HTML blocks with syntax highlighting.
- **FR-10** Render `<span style="color:…">` inline HTML with the specified color.
- **FR-11** Render GFM tables with column-aligned borders and distinct cell background.

### Navigation
- **FR-12** Scroll the document with arrow keys, Page Up/Down, Home/End.
- **FR-13** Tab/Shift-Tab cycles focus between `<details>` blocks, scrolling the focused block to the upper third of the viewport.
- **FR-14** Enter toggles the focused `<details>` block open/closed.
- **FR-15** Mouse click on a `<details>` summary toggles it.
- **FR-16** `q` or `Q` quits the application.
- **FR-17** `-` collapses all `<details>` sections at once.
- **FR-18** `=` expands all `<details>` sections at once.

### Text Selection & Copy
- **FR-23** The user can select text by clicking and dragging the mouse.
- **FR-24** Right-clicking copies the selected text to the system clipboard and clears the selection.
- **FR-25** `Ctrl+C` copies the selected text to the system clipboard and clears the selection.
- **FR-26** Selected text is highlighted with inverted foreground/background colors while the selection is active.

### Theming
- **FR-17** Support at least 10 named color themes.
- **FR-18** `t`/`T` cycles forward/backward through available themes.
- **FR-19** Theme change updates all colors including the Toplevel background.

### Unicode
- **FR-20** All cursor tracking and text truncation must use display-column width, not UTF-16 code unit count.
- **FR-21** Emoji with VS16 (U+FE0F) variation selector must be treated as 2 columns wide.
- **FR-22** Non-breaking space (U+00A0) must render as a regular space.

## Non-Functional Requirements

- **NFR-01 Security:** Do not execute any content from the Markdown file. HTML is allowed only for `<details>`, `<summary>`, `<pre><code>`, and `<span style="">` elements.
- **NFR-02 Maintainability:** Clean, modular code. Files must not exceed 300 lines; refactor if they do.
- **NFR-03 User Experience:** Beautiful, aesthetic UI with smooth scrolling and clear visual hierarchy.
- **NFR-04 Performance:** Only visible nodes are rendered per draw cycle; no full-document re-render on every frame.
